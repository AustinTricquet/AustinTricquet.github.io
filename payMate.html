The Problem

What a mess!

Contracted prison labor payroll system was insecure, confusing to understand and operate in a timely manner. Tasked with simplifing the process and insuring the payroll system accuracy, while maintaining a weekly payroll for 300+ employees.Fun Fact: The old payroll Excel file ranged from A - IG! - That's a lot of alphabets!

Initial Findings:

  • Confusing and complicated system based in Microsoft Excel.
  • Low accountability for payment accuracy.
  • Insecure employee data.
  • Few with the experience or understanding to maintain the system.

Solution I

Making sparks with Tkinter

Once I saw the oppurtunity to improve this process, I challenged myself to learn programming. I quickly discovered that programming is my passion. Since I already had some basic understanding of Python, That's where I started to learn more about Python and to provide a solution to this problem.Fun Fact: This is my first real programming project, GUI included! - Not too bad!

Solution I:

  • Technologies and tools used: Python, Tkinter, Microsoft Access & Excel, Visual Studio
  • Access database already in use, exports a pay roster to excel which the app can access.
  • GUI created with Tkinter, used to enter hours for each employee instead of an Excel sheet.

Problems with Solution I:

  • Limited programming experience led to poor design of code which became an issue in expanding the software to include additional departments and shift codes used in calculations.
  • Tkinter limitations and UI layout was poorly designed and need a more modern implementation.
  • Fixing bugs and issues with deployment, became increasingly difficult due to poor code design.

Solution II

Django was his name-o

By taking online classes, I gained further experience programming and understanding better design principles in code. The flaws in the first solution were evident. For the next solution I chose to use Django given my experience with Python and the ability to provide a better UI experience than Tkinter.Fun Fact: Using tools like Django made adding proper User Authentication easy!

Solution I:

  • Technologies and tools used: Django, Python, HTML, CSS, Javascript, JQuery, Bootstrap 4, PostgreSQL, VSCode, Git & Github
  • Used Git and Github for version control and logging updates on progress during development.
  • Added new features including: proper user Authentication, User permissions, Admin panel to add users, change user permissions, edit employee information, Add and remove departments and work locations etc.
  • Overhaul of UI look and layout using HTML, CSS, Javascript, JQuery, and Bootstrap 4

Checkout the Github or try the PayMate Demo below!

Fun Fact: Each employee has their own card with hours and other data presented. Each person can be searched for by name or ID number.
