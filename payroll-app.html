The Problem

Payroll system used inside a correctional facility was insecure, confusing to understand and operate in a timely manner. Tasked with simplifing the process and insuring the payroll system accuracy, while maintaining a weekly payroll for 300+ employees.

Initial Findings:

  • Confusing and complicated system based in Microsoft Excel.
  • Low accountability for payment accuracy.
  • Insecure employee data.
  • Few with the experience or understanding to maintain the system.

Solution

I chose to use Django given my experience with Python from my classes at Auburn University. I liked the features that Django offered with easy to impliment authentication and admin controls.

  • Technologies and tools used: Django, Python, HTML, CSS, Javascript, JQuery, Bootstrap 4, PostgreSQL, VSCode, Git & Github.
  • Added user Authentication, User permissions, Admin panel to add users, change user permissions, edit employee information, Add and remove departments and work locations etc.
  • Leveraged Bootstrap 4 for a easy to impliment and modern looking UI.
  • Used a local PostgreSQL server to store previous payroll information and to autosave progress on each new data entry.
  • Unit tested all functions to add an extra level of confidence in the integrity of the application.
