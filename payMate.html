PayMate

Weekly payroll and financial reporting software

  • Github - Coming soon!
  • Demo - Coming soon!

The Problem: What a mess!

Contracted prison labor payroll system was insecure, confusing to understand and operate the payroll in a timely manner. Tasked with simplifing the process and insuring the payroll system accuracy.

Initial Findings:

  • - Very Confusing and complicated system based in Microsoft Excel.
  • - Low accountability for system accuracy and insecure employee information.
  • - Few with the experience or understanding to maintain the system.

Solution 1: Making sparks with Tkinter

Once I saw the oppurtunity to improve this process, I challenged myself to learn programming. I quickly discovered that programming is my passion. Since I already had some basic understanding of Python, That's where I started to learn more about Python and to provide a solution to this problem.

Solution 1:

  • - Technologies and tools used: Python, Tkinter, Microsoft Access & Excel, Visual Studio
  • - Access database already in use, exports a pay roster to excel which the app can access.
  • - GUI created with Tkinter, used to enter hours for each employee instead of an Excel sheet.

Problems with Solution 1:

  • - Limited programming experience led to poor design of code which became an issue in expanding the software to include additional departments and shift codes used in calculations.
  • - Tkinter limitations and UI layout was poorly designed and need a more modern implementation.
  • - Fixing bugs and issues with deployment, became increasingly difficult due to poor code design.

Solution 2: Django was his name-o

By taking online classes, I gained further experience programming and understanding better design principles in code. The flaws in the first solution were evident. For the next solution I chose to use Django given my experience with Python and the ability to provide a better UI experience than Tkinter.

Solution 2:

  • - Technologies and tools used: Django, Python, HTML, CSS, Javascript, JQuery, Bootstrap 4, PostgreSQL, VSCode, Git & Github
  • - Used Git and Github for version control and logging updates on progress during development.
  • - Added new features including: proper user Authentication, User permissions, Admin panel to add users, change user permissions, edit employee information, Add and remove departments and work locations etc.
